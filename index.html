Projects

Rpopular100 Reddit Client

This project was created as part of the Codecademy Fullstack career path. The goal was to create a Reddit client using React and Redux. Instead of making a clone of Reddit, I decided to make a lightweight reddit client that only shows the top 100 posts from r/popular.

The app was made using create-react-app with a react-redux template. The app pulls data from reddit's "JSON API", adds it to state, and renders it.

The app isn't finished, but the basic functionality is there. If I have the time, I'd like to add a few features, such as infinite scrolling and a way to block posts from specific subreddits from being displayed. The styling of the app could also use a lot of work.

Shopify Backend Developer Intern Challenge - Fall 2022

This project was created as part of my application for a backend development internship at Shopify. The goal of the project was to create a functioning inventory management app for a fictional logistics company. The app had to be able to perform basic CRUD functions, as well as one additional piece of functionality. In my case, I decided to implement functionality that allows the user to submit deletion comments when deleting an item from inventory, as well as being able to undo deleting an item.

The backend of the app is a Spring Boot REST API that uses Spring Data JPA to perform CRUD operations on an in-memory H2 database. The frontend is a simple series of static pages that use JS to communicate with the backend.

I decided to use Spring even though I wasn't that familiar with it at the time because I felt that it was the cleanest and most modern way to approach the problem using Java. It turned out to be much more challenging than I originally thought it would be, but I'm glad I stuck with it. Spring is a great framework that abstracts away much of the "plumbing" underneath a modern web application, but in my experience you still need to have some idea of what's going on underneath the hood. Building this app taught me a lot about REST APIs, Spring and some of its quirks, and even some new things on the frontend, like asynchronous JavaScript functions.

Home nVentory Inventory Management System

This is my final project for SAIT's Web Application Development course. The goal was to build an inventory management web app that would allow users to:

  • sign up
  • sign in and out
  • change their account details
  • add, edit, and remove inventory items
  • deactivate their account

The app also allows for admin users, who can add, edit, and remove users and categories. The app also sends emails, uses filters for authentication, and salts and hashes passwords. The app is built using Java EE, Maven, JPA, JSTL/JSP, and MariaDB/MySQL.

Even though Java EE is a bit older and not that popular these days, I liked building this app because it forced me to do many of the things that would normally be abstracted away by a more modern framework. I learned a fair bit about how an ORM like JPA interacts with a database and about mapping objects and classes to database entities. As always, building a project like this teaches you a lot, because when it comes to actually implementing the theory you feel you already know, you inevitably run into stumbling blocks that you have to work through or around to get the system to work properly.
